# Review Checklist

Use this checklist while extending tesyt4.

- [ ] `README.md` still explains how to run the project.
- [ ] `project-summary.md` still matches selected modules.
- [ ] `.env` is ignored and real secrets are not committed.
- [ ] `env.example` contains placeholders only.
- [ ] Access control is reviewed where sensitive data is processed.
- [ ] Payment workflows are validated server-side where relevant.
- [ ] Provider documentation is checked before production use.
- [ ] Tests are added for risky workflows.
